# Pruebas Sprint 1.6B

1. Abra `frontend/index.html` mediante GitHub Pages o un servidor local.
2. Ingrese a **Sincronización**.
3. Confirme el ID precargado.
4. Pulse **Vista previa**.
5. Revise registros y observaciones.
6. Pulse **Importar datos**.
7. Abra Dashboard, Territorios, Acciones, Indicadores y Reportes.
8. Confirme que los datos importados sustituyeron los datos simulados.
9. Recargue la página y confirme que la caché persiste.
10. Revise el historial local de sincronización.

Nota: abrir `index.html` directamente puede estar limitado por políticas CORS del navegador. Para pruebas confiables use GitHub Pages o un servidor local, por ejemplo `python -m http.server 8000`.
